\doxysubsection{Detailed Description}
High-\/level pulsar object that bundles structure, spin, thermal, and BNV state. 

Comprehensive pulsar representation with rotation, structure, composition, and thermal evolution.\hypertarget{class_compact_star_1_1_pulsar_autotoc_md0}{}\doxyparagraph{Responsibilities}\label{class_compact_star_1_1_pulsar_autotoc_md0}

\begin{DoxyItemize}
\item Owns (or imports) a {\bfseries{structural profile}} ({\ttfamily \mbox{\hyperlink{struct_compact_star_1_1_star_profile}{Star\+Profile}}}) and exposes it as a view.
\item Stores {\bfseries{observed spin parameters}} in a {\ttfamily Spin\+State}.
\item Stores {\bfseries{thermal parameters}} in a {\ttfamily Thermal\+State}.
\item Stores {\bfseries{BNV diagnostics}} in a {\ttfamily BNVState}.
\item Optionally attaches an {\bfseries{EOS}}.
\item Optionally remembers its {\bfseries{sequence point}} ({\ttfamily \mbox{\hyperlink{class_compact_star_1_1_seq_point}{Seq\+Point}}}) and the parent {\bfseries{sequence profile}} (for mass–radius sequences, evolutionary tracks, etc.).
\end{DoxyItemize}\hypertarget{class_compact_star_1_1_pulsar_autotoc_md1}{}\doxyparagraph{Non-\/\+Responsibilities}\label{class_compact_star_1_1_pulsar_autotoc_md1}

\begin{DoxyItemize}
\item Does {\bfseries{not}} do raw TOV integration (that’s still in {\ttfamily \mbox{\hyperlink{class_compact_star_1_1_t_o_v_solver}{TOVSolver}}}).
\item Does {\bfseries{not}} implement DUrca/\+MUrca formulas directly (that’s in {\ttfamily \mbox{\hyperlink{_thermal_8hpp}{Physics/\+Thermal.\+hpp}}}).
\item Does {\bfseries{not}} hard-\/code plotting.
\end{DoxyItemize}

The idea is to keep this class small and make it the thing your higher-\/level code passes around.

Extends \mbox{\hyperlink{class_compact_star_1_1_prog}{Prog}} for logging and directory management. Handles\+:
\begin{DoxyItemize}
\item Spin and spin-\/down observables.
\item \mbox{\hyperlink{class_compact_star_1_1_baryon}{Baryon}} number violation constraints via spin-\/down energy budget.
\item Structural profiles from TOV solver\+: mass, radius, metric functions.
\item Composition profiles\+: baryon species fractions and Fermi energies.
\item Thermal physics\+: core, blanket, and surface temperatures, photon luminosity.
\item Neutrino cooling\+: DUrca and MUrca processes with physical emissivity prefactors.
\item Plotting utilities for composition, metric, and cooling diagnostics.
\end{DoxyItemize}

See ar\+Xiv\+:2201.\+02637 for definitions of η\+\_\+I and spin-\/down BNV limits.

This replaces the old, inline logic that used to perform file import, interpolation, and DUrca/blanket calculations directly inside \mbox{\hyperlink{class_compact_star_1_1_pulsar}{Pulsar}}. It now delegates those operations to \mbox{\hyperlink{namespace_compact_star_1_1_star_builder_a3b94da4e460dfad279f5cac1c5300b0d}{Compact\+Star\+::\+Star\+Builder\+::\+Build\+From\+Sequence}}, which handles\+:
\begin{DoxyItemize}
\item loading the sequence file {\ttfamily \texorpdfstring{$<$}{<}model\+\_\+name\texorpdfstring{$>$}{>}\+\_\+\+Sequence.\+tsv}
\item identifying the profile index closest to the pulsar mass ({\ttfamily mp})
\item interpolating between neighboring profiles if needed
\item computing the structural point seq\+\_\+point and η\+\_\+I parameter
\item locating the heat-\/blanket radius ({\ttfamily r\+\_\+blanket})
\item optionally building a Direct-\/\+Urca mask ({\ttfamily r\+\_\+durca\+\_\+cond})
\end{DoxyItemize}

The resulting data set is stored in the Pulsar’s internal members for later use by thermodynamic, BNV, or spin modules.
