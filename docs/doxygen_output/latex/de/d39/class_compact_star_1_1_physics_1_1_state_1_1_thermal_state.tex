\doxysubsection{Detailed Description}
Dynamical thermal DOFs + cached physical temperatures. 

The ODE part (values\+\_\+) is what evolution drivers operate on. By default callers usually use a single DOF representing the {\bfseries{redshifted internal temperature}} T∞. If more thermal components (e.\+g. multi-\/zone models) are needed, callers simply call Resize(\+N).

The auxiliary fields store physically meaningful temperatures in the {\bfseries{local fluid frame}}, which microphysics and cooling models often require.

\begin{DoxyNote}{Note}
Storage (the {\ttfamily values\+\_\+} vector) deliberately lives in this derived class rather than in the \mbox{\hyperlink{class_compact_star_1_1_physics_1_1_state_1_1_state}{State}} base class. This design allows future state types to use non-\/contiguous, multi-\/zone, or externally owned layouts while still exposing the uniform Pack\+To/\+Unpack\+From interface required by the evolution framework. See \mbox{\hyperlink{class_compact_star_1_1_physics_1_1_state_1_1_state}{Compact\+Star\+::\+Physics\+::\+State\+::\+State}} for details. 
\end{DoxyNote}
