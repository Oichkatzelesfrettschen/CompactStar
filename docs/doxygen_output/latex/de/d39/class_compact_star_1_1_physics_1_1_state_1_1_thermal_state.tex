\doxysubsection{Detailed Description}
Dynamical thermal DOFs + cached physical temperatures. 

The ODE part ({\ttfamily values\+\_\+}) stores the {\bfseries{thermal evolution variables}} operated on by evolution drivers. In the default (single-\/zone) configuration\+:


\begin{DoxyItemize}
\item {\ttfamily values\+\_\+\mbox{[}0\mbox{]} ≡ ln(T∞ / T\+\_\+ref) (dimensionless)}
\end{DoxyItemize}

where T∞ is the {\bfseries{redshifted isothermal-\/core temperature}} in Kelvin.

Storing the logarithm of the temperature ensures positivity by construction and improves numerical stability over long cooling timescales. Evolution drivers must therefore contribute to\+:

d(ln(T∞/\+T\+\_\+ref))/dt = (1 / T∞) · d\+T∞/dt

rather than directly evolving T∞ itself.

Additional thermal components (e.\+g. multi-\/zone models) can be introduced by calling {\ttfamily Resize(\+N)}; the interpretation of components beyond index 0 is left to higher-\/level thermal models.

The auxiliary fields ({\ttfamily T\+\_\+core}, {\ttfamily T\+\_\+blanket}, {\ttfamily T\+\_\+surf}) store physically meaningful temperatures in the {\bfseries{local fluid frame}}. These are {\bfseries{not}} part of the ODE vector and are intended for use by microphysics, envelope models, cooling laws, and diagnostics.

\begin{DoxyNote}{Note}
Storage (the {\ttfamily values\+\_\+} vector) deliberately lives in this derived class rather than in the {\ttfamily \mbox{\hyperlink{class_compact_star_1_1_physics_1_1_state_1_1_state}{State}}} base class. This design allows future state types to use non-\/contiguous, multi-\/zone, or externally owned layouts while still exposing the uniform Pack\+To/\+Unpack\+From interface required by the evolution framework. See \mbox{\hyperlink{class_compact_star_1_1_physics_1_1_state_1_1_state}{Compact\+Star\+::\+Physics\+::\+State\+::\+State}} for details. 
\end{DoxyNote}
