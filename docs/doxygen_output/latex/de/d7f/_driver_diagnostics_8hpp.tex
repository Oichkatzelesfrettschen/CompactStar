\doxysubsection{Detailed Description}
Optional diagnostics interface for Physics drivers. 

This header defines a small, non-\/intrusive interface that any driver can implement to provide a structured diagnostics snapshot at runtime.

Motivation\+:
\begin{DoxyItemize}
\item Avoid ad-\/hoc debug prints scattered across drivers.
\item Provide deterministic JSON/\+JSONL records for tooling and regression tests.
\item Centralize \char`\"{}unit contract\char`\"{} lines per driver (conventions \& assumptions).
\end{DoxyItemize}

Design constraints\+:
\begin{DoxyItemize}
\item Numeric payload is still plain doubles.
\item Diagnostics are intended to be called rarely (e.\+g., t=0 or every N steps), so they should not influence performance.
\end{DoxyItemize}

Usage\+:
\begin{DoxyItemize}
\item A driver may implement IDriver\+Diagnostics.
\item An observer (Diagnostics\+Observer) queries drivers that implement this interface and writes results to JSONL. 
\end{DoxyItemize}
