\doxysubsection{Detailed Description}
Lightweight read-\/only context passed to evolution drivers. 

Driver\+Context is a {\bfseries{non-\/owning}} bundle of pointers to slowly-\/varying / static objects that drivers may consult when computing RHS contributions.

Design goals\+:
\begin{DoxyItemize}
\item Keep the driver interface explicit and stable.
\item Avoid circular dependencies between drivers and Evolution\+System.
\item Prevent heavy includes in driver-\/facing headers.
\end{DoxyItemize}

Ownership\+:
\begin{DoxyItemize}
\item All pointers are {\bfseries{non-\/owning}} and must remain valid for the duration of the evolution run (typically owned by Evolution\+System or the caller).
\end{DoxyItemize}

Optional components\+:
\begin{DoxyItemize}
\item Some pointers (e.\+g., envelope boundary condition models) may be nullptr. They are {\bfseries{not}} required globally; drivers that require them must validate their presence based on their own Options/configuration. 
\end{DoxyItemize}
