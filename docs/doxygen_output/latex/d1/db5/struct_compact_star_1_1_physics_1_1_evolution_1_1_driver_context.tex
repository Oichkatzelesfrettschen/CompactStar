\doxysubsection{Detailed Description}
Read-\/only static context for evolution drivers. 

\mbox{\hyperlink{struct_compact_star_1_1_physics_1_1_evolution_1_1_driver_context}{Driver\+Context}} bundles together pointers to static model objects that drivers may consult while computing d\+Y/dt.

It is passed {\bfseries{by const reference}} into \mbox{\hyperlink{class_compact_star_1_1_physics_1_1_i_driver_ab4c100ea49a9ec8bad3b43f9f84c17eb}{IDriver\+::\+Accumulate\+RHS()}}.

Key principles\+:
\begin{DoxyItemize}
\item {\bfseries{Non-\/owning}} pointers
\item {\bfseries{Read-\/only}} usage contract for drivers
\item {\bfseries{Optional components}} allowed (nullptr is valid)
\end{DoxyItemize}

Validation policy\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{class_compact_star_1_1_physics_1_1_evolution_1_1_evolution_system}{Evolution\+System}} should validate only globally required pointers (e.\+g. star/geo/cfg depending on your run mode).
\item Feature-\/specific pointers (e.\+g. envelope) must be validated by the driver that uses them, based on driver Options. 
\end{DoxyItemize}
