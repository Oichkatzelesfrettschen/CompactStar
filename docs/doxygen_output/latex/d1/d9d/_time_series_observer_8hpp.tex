\doxysubsection{Detailed Description}
Lightweight, plotting-\/oriented time series recorder for Evolution runs. 

The Time\+Series\+Observer is intentionally distinct from Diagnostics\+Observer\+:


\begin{DoxyItemize}
\item Diagnostics\+Observer is designed for auditing, unit contracts, regression testing, and structured per-\/driver snapshots (JSONL with rich metadata).
\item Time\+Series\+Observer is designed for {\itshape plotting and analysis}\+: it records a narrow set of scalar quantities (e.\+g. evolved DOF like temperature and spin frequency) as a function of time, with minimal overhead and minimal repetition.
\end{DoxyItemize}

Design goals\+:
\begin{DoxyItemize}
\item Deterministic column ordering.
\item Simple, tooling-\/friendly output (CSV by default).
\item No \char`\"{}contract\char`\"{}, warnings, or verbose metadata in each row.
\item Observer must never mutate state or context (read-\/only).
\end{DoxyItemize}

Typical output row\+: t, sample\+\_\+index, Tinf\+\_\+K, Omega\+\_\+rad\+\_\+s, d\+Ln\+Tinf\+\_\+dt\+\_\+1\+\_\+s, d\+Omega\+\_\+dt\+\_\+rad\+\_\+s2, ...

Column selection\+:
\begin{DoxyItemize}
\item The observer supports an explicit list of columns (keys) to record.
\item Columns can be sourced from\+: (a) state-\/only accessors (e.\+g. Thermal\+State\+::\+Tinf()), (b) registered driver diagnostics providers (IDriver\+Diagnostics), (c) future\+: derived/computed columns (via callback hooks).
\end{DoxyItemize}

NOTE\+:
\begin{DoxyItemize}
\item This header defines the interface and configuration only.
\item The implementation (cpp) is expected to\+:
\begin{DoxyItemize}
\item open/close files in On\+Start/\+On\+Finish,
\item write a single header line,
\item write one row per recorded sample,
\item enforce stable ordering and minimal allocations. 
\end{DoxyItemize}
