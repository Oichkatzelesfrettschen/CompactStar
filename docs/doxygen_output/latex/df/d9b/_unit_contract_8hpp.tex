\doxysubsection{Detailed Description}
Runtime “unit contract” utilities for diagnostics. 

This module supports two ideas you discussed\+:

(1) Centralize unit assumptions in one place per driver (runtime contract)\+:
\begin{DoxyItemize}
\item A driver provides a Unit\+Contract with stable text lines.
\item Diagnostics can print the contract verbatim (no duplication of formulas).
\end{DoxyItemize}

(2) Enforce a global allowed unit vocabulary (runtime linting)\+:
\begin{DoxyItemize}
\item You can validate that unit strings used by diagnostics are from a predefined set (e.\+g., \char`\"{}\+K\char`\"{}, \char`\"{}erg/s\char`\"{}, \char`\"{}cm$^\wedge$2\char`\"{}, \char`\"{}rad/s\char`\"{}).
\item This prevents drift like \char`\"{}erg s$^\wedge$-\/1\char`\"{} vs \char`\"{}erg/s\char`\"{} across drivers.
\end{DoxyItemize}

Important\+:
\begin{DoxyItemize}
\item This is NOT a unit type system; it is metadata and linting.
\item Validation should be cheap and used only in debug/diagnostic paths. 
\end{DoxyItemize}
