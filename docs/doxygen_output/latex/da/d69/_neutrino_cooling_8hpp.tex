\doxysubsection{Detailed Description}
Core neutrino-\/cooling driver (DUrca, MUrca, PBF hooks) for thermal evolution. 

\hypertarget{_neutrino_cooling_8hpp_autotoc_md26}{}\doxysubsubsection{Physical intent}\label{_neutrino_cooling_8hpp_autotoc_md26}
This driver contributes a {\bfseries{neutrino luminosity loss}} term to the redshifted thermal degree of freedom (typically (T\+\_\+\textbackslash{}infty)). Compact\+Star evolves a {\bfseries{logarithmic temperature variable}}\+:

\[
  x \equiv \ln\!\left(\frac{T_\infty}{T_{\rm ref}}\right),
\]

so the driver must accumulate (dx/dt) into the thermal RHS\+:

\[
  \frac{dx}{dt} = \frac{1}{T_\infty}\,\frac{dT_\infty}{dt}.
\]

In a full neutrino-\/cooling model one would compute\+:

\[
  L_{\nu,\infty} = \int dV_{\rm proper}\; Q_\nu(r)\times(\text{redshift factors}),
\]  \[
  \frac{dT_\infty}{dt} = -\frac{L_{\nu,\infty}}{C_{\rm eff}},
\]  \[
  \frac{dx}{dt} = -\frac{L_{\nu,\infty}}{C_{\rm eff}\,T_\infty}.
\]\hypertarget{_neutrino_cooling_8hpp_autotoc_md27}{}\doxysubsubsection{Implementation pattern}\label{_neutrino_cooling_8hpp_autotoc_md27}
This driver follows the same architecture as Photon\+Cooling\+:
\begin{DoxyItemize}
\item Physics and diagnostics share the same computation via {\ttfamily Neutrino\+Cooling\+\_\+\+Details}.
\item {\ttfamily Accumulate\+RHS()} calls {\ttfamily Detail\+::\+Compute\+Derived()} and adds only the derived (dx/dt) contribution.
\item {\ttfamily Diagnose\+Snapshot()} calls {\ttfamily Detail\+::\+Diagnose()} so diagnostics cannot drift from the physics implementation.
\end{DoxyItemize}\hypertarget{_neutrino_cooling_8hpp_autotoc_md28}{}\doxysubsubsection{Units policy}\label{_neutrino_cooling_8hpp_autotoc_md28}
