\doxysubsection{Detailed Description}
\begin{DoxyNote}{Note}
Design rationale\+:
\end{DoxyNote}
The \mbox{\hyperlink{class_compact_star_1_1_physics_1_1_state_1_1_state}{State}} base class intentionally does {\itshape not} provide storage (e.\+g. no {\ttfamily std\+::vector\texorpdfstring{$<$}{<}double\texorpdfstring{$>$}{>} values\+\_\+} here). This is deliberate\+:


\begin{DoxyItemize}
\item Some states store a simple contiguous vector (Spin/\+Thermal/\+Chem/\+BNV).
\item Future state types may store grid-\/based data, compressed data, external views, or multi-\/zone structures.
\end{DoxyItemize}

By keeping \mbox{\hyperlink{class_compact_star_1_1_physics_1_1_state_1_1_state}{State}} as a {\itshape pure interface}, the evolution system interacts with all state blocks uniformly\+: \begin{DoxyVerb} Size()        → number of DOFs
 Data()        → contiguous pointer
 PackTo()      → flatten into global y[]
 UnpackFrom()  → restore from global y[]
\end{DoxyVerb}
 Each derived class is free to choose its internal representation as long as it exposes the contiguous view required by the evolution framework.

This keeps the core evolution machinery flexible and future-\/proof, while allowing physics modules to use whatever internal layout they need.
