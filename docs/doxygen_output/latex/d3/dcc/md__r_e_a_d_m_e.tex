\hypertarget{md__r_e_a_d_m_e_autotoc_md51}{}\doxysection{Design Philosophy}\label{md__r_e_a_d_m_e_autotoc_md51}
Compact\+Star is built around the following principles\+:


\begin{DoxyItemize}
\item {\bfseries{Physics-\/first abstractions}}\+: physical subsystems (spin, thermal, chemical, BNV) are modeled explicitly and evolve independently via tagged state blocks.
\item {\bfseries{Strict separation of concerns}}\+:
\begin{DoxyItemize}
\item Core\+: structure, EOS, geometry, datasets
\item Physics/\+State\+: degrees of freedom
\item Physics/\+Driver\+: RHS contributions
\item Evolution\+: integration, diagnostics, orchestration
\end{DoxyItemize}
\item {\bfseries{Schema-\/driven diagnostics}}\+: diagnostics are self-\/describing, versioned, and machine-\/readable.
\item {\bfseries{No hidden global state}}\+: all evolution is driven through explicit context objects.
\end{DoxyItemize}

\hypertarget{md__r_e_a_d_m_e_autotoc_md53}{}\doxysection{Drivers vs Microphysics}\label{md__r_e_a_d_m_e_autotoc_md53}
Compact\+Star distinguishes between\+:


\begin{DoxyItemize}
\item {\bfseries{Microphysics}}\+:
\begin{DoxyItemize}
\item reaction rates
\item matrix elements
\item particle models
\item EOS-\/level inputs
\end{DoxyItemize}
\item {\bfseries{Drivers}}\+:
\begin{DoxyItemize}
\item convert microphysics into time derivatives
\item operate on State blocks
\item are stateless apart from configuration
\item are pluggable and composable
\end{DoxyItemize}
\end{DoxyItemize}

Microphysics never modifies state directly. All time evolution enters the system exclusively through Drivers.

\hypertarget{md__r_e_a_d_m_e_autotoc_md68}{}\doxysection{Diagnostics and Provenance}\label{md__r_e_a_d_m_e_autotoc_md68}
Compact\+Star treats diagnostics as first-\/class, schema-\/defined data products.

Every physical quantity emitted during evolution is\+:


\begin{DoxyItemize}
\item explicitly named,
\item unit-\/labeled,
\item source-\/typed ({\ttfamily state}, {\ttfamily computed}, {\ttfamily cache}),
\item cadence-\/controlled (always, on change, once per run),
\item versioned via a diagnostics catalog.
\end{DoxyItemize}

Diagnostics are emitted in three complementary forms\+:


\begin{DoxyItemize}
\item {\bfseries{JSONL packets}} ~\newline
 Full-\/fidelity diagnostic records, suitable for post-\/processing, validation, and long-\/term archival.
\item {\bfseries{Schema catalogs (JSON)}} ~\newline
 Machine-\/readable descriptions of all diagnostics, including units, descriptions, and provenance. These act as a contract between physics drivers and observers.
\item {\bfseries{Compact time-\/series tables (CSV/\+TSV)}} ~\newline
 Plot-\/friendly, schema-\/driven tables derived directly from the catalog, guaranteeing consistency between raw diagnostics and reduced outputs.
\end{DoxyItemize}

This design ensures\+:


\begin{DoxyItemize}
\item reproducibility across runs and code versions,
\item safe post-\/processing without hard-\/coded column assumptions,
\item clear provenance for every reported physical quantity.
\end{DoxyItemize}

\hypertarget{md__r_e_a_d_m_e_autotoc_md72}{}\doxysection{State Interface Architecture}\label{md__r_e_a_d_m_e_autotoc_md72}
Compact\+Starâ€™s evolution engine treats each physical subsystem ~\newline
 (Spin, Thermal, Chem, BNV, â€¦) as a {\bfseries{state block}} derived from a single abstract base class {\ttfamily State}. This base class defines a small, uniform interface\+:


\begin{DoxyItemize}
\item {\ttfamily Size()}
\item {\ttfamily Data()}
\item {\ttfamily \mbox{\hyperlink{_chem_state_8hpp_a3f6089e2db736965f021e2165ae39933}{Pack\+To()}}}
\item {\ttfamily \mbox{\hyperlink{_chem_state_8hpp_af4c43758909d4e294d63b7d067eae761}{Unpack\+From()}}}
\item {\ttfamily Resize()}
\item {\ttfamily \mbox{\hyperlink{_chem_state_8hpp_ac2fdf117e3886bbce4baa56a77d3e9cc}{Clear()}}}
\end{DoxyItemize}

but {\bfseries{it intentionally does not define storage}}. ~\newline
 Each derived state type owns its own internal representation.

\hypertarget{md__r_e_a_d_m_e_autotoc_md74}{}\doxysubsection{Why the base class does $<$em$>$not$<$/em$>$ own $<$tt$>$values\+\_\+$<$/tt$>$}\label{md__r_e_a_d_m_e_autotoc_md74}
Different state types have very different physical and numerical needs\+:


\begin{DoxyItemize}
\item {\bfseries{Spin\+State}} stores Î©-\/like variables in a simple vector
\item {\bfseries{Thermal\+State}} may evolve one or many temperature components
\item {\bfseries{Chem\+State}} evolves a vector of Î· chemical imbalances
\item {\bfseries{BNVState}} may evolve anything from 1 to many exotic parameters
\item Future modules may require 2D grids, GPU buffers, or nested structures
\end{DoxyItemize}

If the base class forced a single storage container (e.\+g. {\ttfamily std\+::vector\texorpdfstring{$<$}{<}double\texorpdfstring{$>$}{>}}), it would prevent advanced future models or force awkward workarounds.

Leaving storage fully up to the derived class gives\+:


\begin{DoxyItemize}
\item full flexibility for arbitrary physics models, present and future
\item decoupling between â€œhow data is storedâ€ and â€œhow evolution operatesâ€
\item identical external interface for all states
\item zero-\/cost abstraction\+: the solver only sees contiguous segments via {\ttfamily Data()}
\end{DoxyItemize}

Every state remains free to choose {\bfseries{any internal representation}}, as long as it can supply a contiguous DOF view to the evolution engine.

\hypertarget{md__r_e_a_d_m_e_autotoc_md76}{}\doxysubsection{Packing and Unpacking\+: How Evolution Works}\label{md__r_e_a_d_m_e_autotoc_md76}

\begin{DoxyCode}{0}

Each state block implements\+:


\begin{DoxyItemize}

\end{DoxyItemize}

This enables\+:


\begin{DoxyItemize}

\item clean separation between the physics model and GSL/\+Sundials interfaces
