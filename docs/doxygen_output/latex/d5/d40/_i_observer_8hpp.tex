\doxysubsection{Detailed Description}
Core observer interface for Evolution module. 

Observers are passive callbacks invoked by the integrator / evolution driver to record or export information (time series, checkpoints, diagnostics, logs).

Key design points\+:
\begin{DoxyItemize}
\item Observers see the full composite state (Evolution\+::\+State\+Vector) and the read-\/only runtime context (Evolution\+::\+Driver\+Context).
\item Observers MUST NOT mutate the ODE state or context.
\item Observers are scheduled by the evolution loop (e.\+g., dt\+\_\+save). The interface does not prescribe scheduling logic; it just provides hooks.
\end{DoxyItemize}

Typical call pattern\+: 1) obs-\/\texorpdfstring{$>$}{>}On\+Start(run, Y0, ctx) 2) During integration, at each \char`\"{}save point\char`\"{}\+: obs-\/\texorpdfstring{$>$}{>}On\+Sample(sample, Y, ctx) (optionally, future\+: obs-\/\texorpdfstring{$>$}{>}On\+Step(step, Y, ctx) for every integrator step) 3) obs-\/\texorpdfstring{$>$}{>}On\+Finish(result, Y\+\_\+final, ctx)
