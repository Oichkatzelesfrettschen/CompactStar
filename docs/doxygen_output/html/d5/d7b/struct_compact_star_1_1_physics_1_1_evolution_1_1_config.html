Detailed Description

Configuration for a single evolution run.

This struct is intentionally small and POD-like; it is passed by const reference into EvolutionSystem and GSLIntegrator.

Integrator-related fields

  • stepper : choice of GSL backend (StepperType).
  • rtol, atol : relative/absolute tolerances for adaptive stepping.
  • max_steps : hard cap on total number of internal steps.
  • dt_save : cadence at which we request output samples.

The actual integrator is implemented in GSLIntegrator.cpp and uses these settings to construct a gsl_odeiv2_driver.
