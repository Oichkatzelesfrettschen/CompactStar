Detailed Description

Interface that a driver can implement to expose diagnostics.

This is intentionally orthogonal to IDriver's runtime API:

  • If a driver doesn't implement it, nothing changes.
  • If it does, an observer can call DiagnoseSnapshot() and UnitContract().

Fill a packet with driver-specific diagnostics at (t, Y, ctx).

This method should:

  • only read state and cache (no mutation),
  • be safe even if some caches are missing (log as notes/warnings),
  • prefer calling the same internal helper functions as the physics path to avoid drift between the code and the diagnostics.
Parameters
tSimulation time [same unit as integrator uses].
YComposite state vector (read-only).
